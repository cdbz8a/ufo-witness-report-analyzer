What this is

You paste a UFO sighting report (like what someone typed online) and the model tries to guess the shape label. Triangle, disk, fireball, light, that kind of thing. It also shows how sure it is and a couple keywords.

It's TF-IDF + Naive Bayes on public report data. Not ChatGPT. Not proof of aliens. Just text classification for class.

How it works (roughly)

  • Grab a big public CSV of sighting reports
  • Clean the text (there's some weird HTML junk in there)
  • Keep 8 shapes that show up enough times. After that I'm at about 54k rows
  • Split train/test 80/20
  • TF-IDF turns words into numbers, Naive Bayes picks a shape
  • On the test set I got like 0.57 accuracy and about 0.51 macro-F1

Most reports are labeled “light”, so accuracy alone is kind of cheating. Macro-F1 is more honest because every shape counts more equally.

Why I picked this for a capstone

I wanted something that wasn't a tiny toy dataset. Real text is messy. Also it's multi-class (8 shapes) not just spam/not spam, and the light class is huge so you actually have to think about metrics. Plus I could put a little website up so people can click around.

Stuff that broke / confused me

  • No header on the CSV. First time I loaded it, pandas used the first report as the column names and everything looked cursed. Had to set the column names myself.
  • Weird &#44; things in the text. Turns out that's just a comma in HTML form. Had to decode that or the words get weird.
  • Everything wants to be “light”. Makes sense because most training examples are light. I started only looking at accuracy and it looked better than it was. Macro-F1 and the confusion matrix fixed that.
  • “Three lights” isnt the same as triangle. The model reads words. If nobody says triangle / triangular / craft, it often just says light. Kind of annoying but that's how this model works.
  • “Circle of lights” bug on the website. The real Python model said circle (~70%). The website said light. I had only put the top words for each shape into the browser, so “circle” never hurt the light score, and I was missing bigrams like “circle lights”. Fixed that and now they match better.
  • Hosting. I tried a free Flask host and it was slow / annoying for a demo. So the public site is just static HTML on GitHub Pages.

Limits

Comments are short. Labels are noisy. It's not magic and it's not detecting real craft. If the wording is vague it usually falls back to light.
